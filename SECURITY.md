# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| Latest release on [GitHub Releases](https://github.com/Delexoo/beats/releases/latest) | Yes |
| Older releases | Best effort |

## Reporting a vulnerability

If you discover a security issue in Beats, please **do not** open a public GitHub issue.

Instead, email **teambrilliantt@hotmail.com** with:

- A description of the vulnerability
- Steps to reproduce
- Impact assessment (if known)
- Your contact information (optional)

We aim to acknowledge reports within **72 hours** and will work with you on a fix before public disclosure when appropriate.

## Scope

This policy covers the Beats desktop application, its installer, and the official project website hosted at [delexoo.github.io/beats](https://delexoo.github.io/beats/).

Third-party tools bundled at runtime (yt-dlp, ffmpeg, spotDL, LibVLC) should be reported to their respective projects unless the issue is specific to how Beats invokes them.
